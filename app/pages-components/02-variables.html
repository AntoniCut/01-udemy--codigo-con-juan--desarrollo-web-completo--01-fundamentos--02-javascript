Variables

var, let y const — el alma de tus datos

Las variables son contenedores con nombre que permiten almacenar valores en memoria. Son el bloque más básico de cualquier programa JavaScript: sin ellas no podrías guardar resultados, contar iteraciones ni modelar información.

JavaScript ofrece tres formas de declararlas: var (clásica, con scope de función), let (moderna, con scope de bloque) y const (constante, no reasignable). Conocer las diferencias entre ellas es clave para escribir código predecible y libre de bugs.

Conceptos que verás

📦

var

Declaración clásica con scope de función y hoisting. Permite redeclarar y reasignar, pero su comportamiento puede generar bugs difíciles de rastrear.

🔒

let

Declaración moderna con scope de bloque. Ideal para variables cuyo valor cambia (contadores, acumuladores, estados en bucles).

🔐

const

Declara una referencia constante: no se puede reasignar el identificador, aunque el contenido del objeto/array sí puede mutar.

🔄

Reasignación

let y var permiten reasignar el valor de la variable. const no. Diferencia crucial para datos mutables e inmutables.

🏷️

Estilos de nombrado

camelCase (recomendado), snake_case, PascalCase y lowercase. Convenciones del lenguaje y cuándo usar cada una.

🧠

Buenas prácticas

Usa const por defecto, let solo cuando necesites reasignar, y evita var en código moderno. Reglas de oro para un JS limpio.
