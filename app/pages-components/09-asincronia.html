09

Asincronía

Operaciones no bloqueantes y Fetch API

JavaScript es single-threaded, pero puede manejar tareas que tardan (peticiones HTTP, timers, lecturas de disco) sin bloquear el hilo principal gracias al modelo asíncrono basado en el Event Loop.

Aprenderás try / catch para errores, Promises como abstracción fundamental, async / await para escribir código asíncrono como si fuera síncrono, y la Fetch API para comunicarte con servidores.

Conceptos que verás

🛡️

try / catch / finally

Manejo de errores con bloques try, captura con catch y limpieza con finally. Lanzar errores con throw new Error().

🤝

Promises

El objeto Promise con estados pending, fulfilled y rejected. Encadenamiento con .then() y .catch().

🔔

Notification API

Notificaciones nativas del navegador con new Notification(), pidiendo permiso al usuario y mostrando mensajes del sistema.

⏱️

async / await

Sintaxis moderna para escribir promesas de forma secuencial, manejo de errores con try / catch y funciones async que devuelven promesas automáticamente.

🔗

Promise.all

Ejecutar múltiples promesas en paralelo y esperar a que todas se resuelvan con Promise.all([...]).

🌐

Fetch API

fetch(url) para peticiones HTTP, manejo de response.json(), headers, métodos POST / GET y errores de red.
